# Labels

This file documents the labels used in the repository. Labels help organise
issues, pull requests, and workflows across the Home Assistant Bible & Cinematic
Dashboard project.

The goal is to keep collaboration predictable, structured, and easy to navigate.

---

## Core Labels

### documentation

Used for changes to files in the `docs/` directory or updates to documentation
elsewhere.

### examples

Used for updates to the `config-examples/` folder.

### experiments

Used for work inside the `experiments/` folder.

### assets

Used for additions or changes to images, mockups, or other visual resources.

### repo-structure

Used for changes to folders, metadata, templates, or organisational files.

---

## Issue Labels

### bug

Used when something is broken or not behaving as expected.

### feature

Used for new functionality or additions to the project.

### improvement

Used for refinements, optimisations, or quality-of-life enhancements.

### question

Used when clarification or guidance is needed.

---

## Priority Labels

### priority-high

Used for urgent or time-sensitive work.

### priority-medium

Used for normal workflow items.

### priority-low

Used for non-urgent or nice-to-have tasks.

---

## Status Labels

### in-progress

Used when work has started on an issue or pull request.

### needs-review

Used when a pull request is ready for review.

### blocked

Used when progress cannot continue until another task is resolved.

---

## AI Collaboration Labels

### ai-scope

Used to indicate the conceptual area an AI should focus on.

### ai-clarification

Used when an AI needs more information before proceeding.

### ai-restriction

Used to signal that certain files or domains must not be touched.

---

## Purpose Summary

Labels ensure:

- clear communication
- predictable workflows
- safe multi-AI collaboration
- easier triage and prioritisation
- consistent organisation across the project

This file should be updated whenever new labels are introduced.
